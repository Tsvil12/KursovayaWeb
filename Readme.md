## 1. Тема
Разработка веб-сервиса для ресторана "Паб Русский" с разделением прав для клиентов и администратора.

## 2. Стек технологий (очень сыро)

Backend: Kotlin + Spring Boot, PostgreSQL, Redis.
Frontend: Vue 3 + TypeScript.

## 3. Соответствие требованиям

Требование: Роли (минимум 2)
Реализация: Администратор (управление меню) и Клиент (оформление заказов)

Требование: Сущности (минимум 5)
Реализация: User, Category, Dish, Order, OrderItem

User - пользователи системы (клиенты и админ)
Category - категории меню (супы, салаты, горячее и т.д.)
Dish - блюда (название, цена, описание, категория)
Order - заказы (дата, статус, сумма, какой клиент)
OrderItem - позиции в заказе (какое блюдо, количество, цена на момент заказа)

Требование: Связь между сущностями
Реализация: Одна категория содержит много блюд, один заказ содержит много позиций

Требование: Транзакционный сценарий
Реализация: Оформление заказа - одновременно создается заказ и позиции заказа, проверяется наличие

Требование: Кэширование (Redis)
Реализация: Меню ресторана хранится в кэше, потому что читается часто, но меняется редко

Требование: Фоновый сценарий
Реализация: Автоматическая отмена заказов, которые не оплатили в течение 30 минут

Требование: Безопасность
Реализация: Админские эндпоинты доступны только админам

Требование: Наблюдаемость
Реализация: Логирование

Требование: Frontend (Vue 3)
Реализация: Страницы входа, меню, корзины, профиля и т.д.

## 4. Ключевые эндпоинты

POST /api/auth/login - вход, получение токена (доступно всем)ё
GET /api/public/menu - просмотр меню (доступно всем, кэшируется)
POST /api/client/orders - создание заказа (только для клиентов)
GET /api/client/orders - история заказов клиента (клиенты и админы)
GET /api/admin/dishes - управление блюдами (только для админов)
PUT /api/admin/dishes/1 - редактирование блюда (только для админов)


## 6. Основные сценарии использования

Клиент заходит на сайт, смотрит меню, добавляет блюда в корзину и оформляет заказ . После этого может посмотреть свои заказы в личном кабинете.

Администратор заходит в админку, может добавлять новые блюда, менять цены, убирать позиции из меню и смотреть список всех заказов.
